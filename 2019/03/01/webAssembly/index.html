WebAssembly

W3C Community Group 制定规范

高效，安全，开放，标准

背景

Atwood 定律，填它的性能坑以让js实现更多的应用

  1. google的JIT在编译器层面处理
  2. Typescript，JSX提前进行静态类型检查
  3. Dart2.0 强类型语言运行在dartVM上
  4. 火狐的 Asm.js 为代表的， 做一个 javascript 子集

但是这些都不是一个标准方案，所以几个主要的浏览器厂商推出了借鉴asm.js的WebAssembly

为什么速度更快

  1. 使用二进制体积更小，下载更快
  2. 解析更快，只做解码和确认代码是否有误
  3. 编译和优化，由于是静态类型，避免类型检查
  4. 没有重优化阶段，不需重新做类型检查
  5. 执行更快，低级的类汇编语言，更好的使用CPU 特性
  6. 垃圾回收，由开发者控制内存的分配和回收

可移植

  • WebAssembly提供字节码，由浏览器编译为机器码，以在各个机器上运行

什么时候使用

  • 对性能有很高要求的App/Module/游戏
  • 在Web中使用C/C++/Rust/Go的库，如ffpemg，压缩程序
  • React dom diff

未来

  • 直接操作 DOM
  • 共享内存的并发性
  • SIMD（单指令，多数据）
  • 异常处理
  • 一流的开发者工具
  • GC
  • ES6模块集成，<script src=url type=“module”>
